searchTVL查询（AND）

searchTVL是自定义标签可以解析的一种查询条件格式。

参数说明：

newsClass：类名，可以理解为表名

page：分页控制中的页面，查询第几页。

pageSize：每页的数据条数。

searchTVL： where条件，多个条件使用逗号(多个逗号不会出错)连接，实际执行时会使用AND连接符连接。val两端没有单引号的，系统会自动加单引号把val的值引起来。

在同一个条件中，在tag、val、link中添加多组数据，可以实现OR查询条件。tag、val、link中的多个值用分号连接，数量必须匹配。

LITTLER = "!{LL}";

BIGGER = "!{BG}";

BIGGER_EQU = "!{BE}";

LITTLE_EQU = "!{LE}";

NOT_EQU = "!{NE}";

EQU = "!{EQ}";

IN = "!{IN}";

LIKE = "!{LIKE}"; 使用like连接符时，如果val传值中没有%，系统会自动在两端补充%

orderName：排序字段.如果有多个用逗号分隔，并且，除了最后一个可以使用扩展名，其他必须用实际的数据库字段名并加上升降序。

如：${idNewsClass.extKeepName.cstartTime} desc,cdate

descOrAsc：升降序，true为降序，false为升序

例如：在colMonitor的表中检索多条数据，检索条件：字段cmonitorCode = 变量monitorCode的值。

Copy

list属性：保存数据库返回的数据列表，通过下边格式可以循环访问

page.totalRecords： 数据库中的总记录条数，不是list的长度

page.pageCount： 总页数

page.pagePrev： 前一页的页码

使用Macro更新

使用封装的宏更新或者复制数据。通常更新逻辑封装成controls，使用ajax调用。

ajax_set_field1_fun提供多个版本，如：ajax_set_field2_fun，可以一次设置2个数据域。

Jsp.call参数说明：

第一个参数： 预定义的jsp文件，在这里可以是ajax_set_field1_fun，ajax_set_field2_fun，ajax_set_field3_fun.....

第二个参数： 更新对象

第三个参数以后，每两个为一组。组内，第一个是要更新的字段名称，第二个是要更新的字段内容。

valueToHBean：更新非扩展字段（非.ext.的字段），只能使用valueToHBean方法。

比如通过将id设置为0，实现同表内的数据拷贝。

_object： 更新对象

_field： 要更新的字段名称

_value： 要更新的字段内容。
